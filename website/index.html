How it works

Three steps. No accounts. No keys. No data leaves your machine.

1

Install

npm i -g @agenttrace-io/cli or pip install agenttrace-io. Create a local DB with agenttrace-io init.

2

Wrap

Wrap your LLM or tool calls with agent.trace(). Or drop in LangGraph / CrewAI middleware for zero-code tracing.

3

Observe

Run agenttrace-io dashboard. See costs, traces, trees, and live usage at http://localhost:4317.

Zero-config self-tracing

Drop in the SDK. Wrap calls with trace(). Automatically captures tokens, cost, latency, tool calls, and errors. Works with any LLM or framework.

Per-agent token & cost tracking

Accurate cost attribution by model with 15+ built-in rates (GPT, Claude, Gemini, Llama…). Register custom rates at runtime. Daily + model breakdowns.

Multi-agent correlation

Trace across sub-agent trees. createChild(), linkTraces(), and getTraceTree() give you the full execution graph for complex agent systems.

Budget alerts & anomaly detection

Register custom alert conditions on stats. Auto-fire webhooks on threshold breach or error spikes. Cooldowns and full delivery history included.

Local SQLite — zero cloud

Everything lives in a single agenttrace.db. Full data ownership. Works offline. Export to JSON, CSV, or OTLP anytime. Retention policies built-in.

Frequently asked questions

Does any data leave my machine?▾
No. All traces, costs, and usage records live in a local SQLite file you control. Exports and webhooks are explicit opt-ins.
How do I instrument an existing agent?▾
Wrap calls with the SDK trace() helper, or use the LangGraph / CrewAI middleware packages for automatic instrumentation. See quickstart.
Can I use this in production?▾
Yes. The core is MIT, fully featured, and used in production by teams that need cost visibility and audit trails without sending data to a third party.
What about LangGraph / CrewAI / custom frameworks?▾
We ship thin middleware for LangGraph and CrewAI that auto-traces. For anything else the SDK works with any async function and any LLM client.
How do I set a daily budget alert?▾
Use registerAlert() in the SDK or the budget CLI commands. Alerts can fire webhooks when thresholds are crossed.
