Frequently asked questions

Is the hosted version using the same SDKs as the open-source project?

Yes. The TypeScript and Python SDKs are identical. Point them at your hosted workspace by passing an API key or endpoint — no code changes required in most cases.

What data gets sent to AgentTrace Hosted?

Only the data you explicitly capture inside trace() calls: prompts, completions, tool calls, token counts, latency, cost metadata, and any custom attributes you attach. We never receive surrounding code or files unless you include them.

Can I stay on the free plan forever?

Absolutely. The free tier is designed to stay useful for development, prototyping, and small personal projects. Upgrade only when you need more volume or team features.

How do I migrate traces from local to hosted?

Use the built-in export (CLI or dashboard) to JSON/OTLP, then import via the hosted API or UI. Many teams run both in parallel during transition.

Do you train models on my agent traces?

No. Your traces are private to your workspace. We do not use customer data for training or product improvement.

Is there a self-hosted option for teams who can't use the cloud?

Yes — the entire open-source project is MIT licensed and runs great on your own infrastructure (Docker, Railway, Render, bare VM). The hosted product is an optional convenience layer.

What frameworks and languages are supported?

Framework-agnostic by design. Native SDKs for TypeScript and Python. Official middleware for LangGraph and CrewAI. OpenTelemetry export works with any OTLP-compatible tool.
