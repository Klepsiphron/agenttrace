# AgentTrace Demo GIF Creation Guide

This document describes how to record a high-quality demo.gif showing the core AgentTrace workflow.

Target dimensions (recommended):
- 960x540 or 1280x720 (16:9) for GitHub README / docs
- Keep under 5-8 seconds for best engagement
- Dark theme only (matches #0d1117 / #38bdf8)

## Recommended Tool: VHS (Charmbracelet)

VHS produces crisp, reproducible terminal GIFs from a simple script. Install once:

```bash
# macOS
brew install vhs

# Linux (see https://github.com/charmbracelet/vhs)
# or go install github.com/charmbracelet/vhs@latest
```

Alternative (manual):
- asciinema + agg (agg --theme monokai or custom)
- OBS Studio + gifski (for dashboard + terminal side-by-side)
- terminalizer (npm i -g terminalizer)

## Step-by-Step Recording Script (VHS .tape)

Create `demo.tape` in a temp folder:

```tape
Output docs/assets/demo.gif
Set Width 960
Set Height 540
Set FontSize 14
Set Padding 12
Set Margin 8
Set Theme "Monokai"   # or "Dracula", custom dark works well
Set Shell "bash"
Set TypingSpeed 28ms
Set LoopOffset 0

# 1. Clean start
Type "mkdir -p /tmp/agenttrace-demo && cd /tmp/agenttrace-demo" Sleep 600ms  Enter
Type "rm -f agenttrace.db demo.js 2>/dev/null || true" Sleep 300ms  Enter
Sleep 400ms

# 2. Install
Type "echo '$ npm install @agenttrace/sdk'" Sleep 300ms  Enter
Type "npm install @agenttrace/sdk --silent" Sleep 1800ms  Enter
Sleep 600ms

# 3. Create a realistic demo script
Type "cat > demo.js << 'EOF'" Sleep 200ms  Enter
Type "import { init, trace } from '@agenttrace/sdk';" Sleep 180ms  Enter
Type "" Sleep 80ms  Enter
Type "const agent = init({ dbPath: './agenttrace.db' });" Sleep 180ms  Enter
Type "" Sleep 60ms  Enter
Type "async function researchAgent(query) {" Sleep 120ms  Enter
Type "  return await trace('research', async () => {" Sleep 120ms  Enter
Type "    // Simulate LLM + tool calls" Sleep 120ms  Enter
Type "    await trace('search', () => searchDocs(query), { model: 'gpt-4o-mini', tokens: { prompt: 180, completion: 42 } });" Sleep 220ms  Enter
Type "    await trace('summarize', () => summarize(), { model: 'claude-3-haiku', tokens: { prompt: 320, completion: 95 } });" Sleep 220ms  Enter
Type "    return { answer: 'Traced successfully', cost: 0.0023 };" Sleep 120ms  Enter
Type "  }, { tags: ['demo'] });" Sleep 120ms  Enter
Type "}" Sleep 120ms  Enter
Type "" Sleep 60ms  Enter
Type "function searchDocs(q) { return 'results for ' + q; }" Sleep 140ms  Enter
Type "function summarize() { return 'concise summary'; }" Sleep 140ms  Enter
Type "" Sleep 60ms  Enter
Type "await researchAgent('agent observability');" Sleep 120ms  Enter
Type "console.log('Done. Run: npx agenttrace dashboard');" Sleep 120ms  Enter
Type "EOF" Sleep 200ms  Enter
Sleep 500ms

# 4. Show the code (cat for viewer)
Type "cat demo.js" Sleep 1200ms  Enter
Sleep 900ms

# 5. Run the traced agent (produces data)
Type "node --input-type=module demo.js" Sleep 400ms  Enter
Sleep 2200ms

# 6. Show CLI results (traces captured)
Type "echo '$ npx agenttrace runs --limit 5'" Sleep 280ms  Enter
Type "npx agenttrace runs --limit 5" Sleep 400ms  Enter
Sleep 1600ms

Type "echo '$ npx agenttrace stats'" Sleep 280ms  Enter
Type "npx agenttrace stats" Sleep 400ms  Enter
Sleep 1400ms

# 7. Launch dashboard (non-blocking note for viewer)
Type "echo 'Starting dashboard on http://localhost:3456 ... (Ctrl+C to stop)'" Sleep 300ms  Enter
Sleep 900ms

# Optional: if you want dashboard visible in same recording,
# run in split pane or record a second pass that opens browser.
# For pure terminal demo this is sufficient.

# End with clean prompt
Type "cd - >/dev/null && rm -rf /tmp/agenttrace-demo" Sleep 200ms  Enter
```

## How to Render the GIF

```bash
vhs demo.tape
```

Result lands at `docs/assets/demo.gif` (or wherever Output points).

## Manual / Fallback Recording Steps (no VHS)

1. Terminal 1 (left or top):
   ```bash
   mkdir -p /tmp/agenttrace-demo && cd /tmp/agenttrace-demo
   npm install @agenttrace/sdk
   ```

2. Create `demo.js` (copy the JS from the tape above or simpler version):

   ```js
   import { init, trace } from '@agenttrace/sdk';
   const agent = init({ dbPath: './agenttrace.db' });

   async function runDemo() {
     await trace('research', async () => {
       await trace('search', async () => ({ hits: 12 }), { model: 'gpt-4o-mini', tokens: { prompt: 142, completion: 28 } });
       await trace('analyze', async () => 'ok', { model: 'claude-3-haiku', tokens: { prompt: 260, completion: 74 } });
       return 'complete';
     });
     console.log('✅ Traces written to agenttrace.db');
   }
   runDemo();
   ```

3. Run:
   ```bash
   node --input-type=module demo.js
   ```

4. Inspect:
   ```bash
   npx agenttrace runs
   npx agenttrace stats
   ```

5. Dashboard (second terminal pane or side-by-side recording):
   ```bash
   npx agenttrace dashboard
   ```
   - Open http://localhost:3456 in browser
   - Click a run to expand traces
   - Show cost, latency, tokens, status pills

6. Record:
   - Use OBS (record region exactly the terminals + browser)
   - Or `peek` / `simplescreenrecorder`
   - Trim + convert to GIF with:
     ```bash
     ffmpeg -i screen.mp4 -vf "fps=12,scale=960:-1:flags=lanczos" -c:v gif demo.gif
     ```
     or gifski:
     ```bash
     gifski --fps 12 --width 960 -o demo.gif frames/*.png
     ```

## Visual Beats to Capture (in order)

- Clean `npm install` (shows zero-config)
- Small but realistic code example (trace() wrapping async work + options)
- Execution of the script (traces written)
- `agenttrace runs` output (table of runs)
- `agenttrace stats` (shows cost, tokens, latency)
- Dashboard starting + browser view:
  - Stats cards
  - Run list with status
  - Expanding a run → individual traces with tool/LLM details
- Optional: filter buttons, export buttons (but keep short)

## Tips for Quality

- Record at 2x resolution then downscale for sharpness.
- Use consistent typing speed (not instant paste).
- Keep total runtime 6–9s.
- End on the dashboard or a clean `npx agenttrace runs` table.
- After recording, optimize:
  ```bash
  # gifsicle (lossless-ish)
  gifsicle -O3 --colors 256 demo.gif -o demo.gif
  ```
- Update README.md / docs-site to reference the new GIF when ready.

## Files Involved

- `docs/assets/demo.gif` (the final asset, committed)
- `docs/assets/demo.gif-description.txt` (this file — source of truth for regeneration)

Run `vhs docs/assets/demo.tape` anytime the demo needs refreshing after major UX changes.
